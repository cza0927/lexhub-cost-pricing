01 · Business context

项目定位与合作价值

LexHub 不是单一“翻译工具”或“合同审查插件”，而是面向中俄及中亚跨境法律业务的双语工作操作系统：让两种语言的用户在同一个案件、同一份文件、同一条任务链中协作。

目标客户与核心场景

中国律所与企业法务

  • 看懂俄语/当地语言合同与来函
  • 快速定位准据法、争议解决与履约风险
  • 把中文意见安全转换为俄文交付

俄语区律所与企业

  • 以全俄文界面接收中国客户材料
  • 自动获得中文材料的俄文语义版本
  • 与中国团队共享文件、消息和任务

跨境项目与仲裁团队

  • 管理多方、多语言、多版本文件
  • 提取争议事实、请求、金额与时限
  • 生成双语工作底稿并保留证据链

价值主张

所有效率指标都必须建立在“法律专家复核、来源可追溯、翻译可对齐”的前提上。

价值维度当前痛点系统带来的改变验证方式

专业质量通用模型不懂当地法域与审查规则国家 Agent 只在已确认法源与 playbook 内工作50 份金标 + 500 份回归集

可控风险译文和 AI 结论来源不清原文锚点、法源引用、版本、模型与人工确认全留痕审计抽查与高风险强制复核率

02 · Product blueprint

产品能力全景

产品由“法律专业能力、双语协同能力、律师日常管理能力”三层组成。每项功能都必须落到案件、客户、文件和责任人上，而不是孤立的聊天回答。

核心法律业务功能

核心 01

合同审查

解析 PDF/Word/扫描件；识别主体、金额、期限、准据法与争议条款；按国家 playbook 扫描风险；输出高/中/低等级、原文锚点、法源依据、修改建议与双语报告。

核心 02

合同生成

通过结构化问答收集交易要素，调用国家模板与条款库生成中文、俄文或双语合同；对关键术语、主体、数字和日期做一致性校验，并保留版本差异。

核心 03

仲裁辅助

提取当事人、争议金额、仲裁请求、事实与理由；整理证据时间线；匹配经专家确认的规则库；生成申请书/答辩思路双语底稿和待律师确认事项。

核心 04

法律检索与分析

按法域、效力层级、发布日期与主题检索；回答必须引用已入库法源，显示原文、译文、有效性状态与检索时间；支持跨国规则对比但不混用法域。

辅助与管理功能

这些是产品范围；具体软件实现方式留给研发合伙人填写。

双语消息与文件

中文用户发中文，俄语用户收俄文；双方可查看原文与译文、修订与确认状态。

邮件处理

邮件归档到客户/案件，摘要、翻译、附件识别、回复草稿与待办抽取。

智能提醒与排期

从合同、邮件和案件中识别履约节点、时限与会议，形成双语提醒。

任务管理

任务与文件、消息、审查结论关联；支持责任人、截止日期、状态与审计记录。

工作空间

律所/企业/项目独立空间，客户与案件数据隔离，权限按角色和事项控制。

报告与知识沉淀

审查报告、律师修订、术语、模板与案例反馈沉淀为可复用知识资产。

03 · Bilingual-by-design

中俄双语系统：不是翻译按钮，而是双语工作环境

系统维护一份事实与业务状态，以中文和俄文两套完整界面呈现。俄语用户看到菜单、字段、系统提示、任务、消息、文件摘要和 Agent 结果的俄文版本；中国用户看到对应中文版本。

UI 全量双语
菜单、字段、错误、通知、帮助与模板
内容即时转换
消息、邮件、任务、批注、文件与报告
法律语义保真
术语、主体、金额、日期、引用与格式锁定

一条内容的双语处理链

1. 输入识别消息/邮件/文件/OCR/语音，识别语言与法域
2. 结构抽取主体、金额、日期、条款、引用、任务与意图
3. 术语锁定调用国家术语库，禁止关键实体漂移
4. 语义转换按法律文本/日常沟通选择不同策略
5. 一致性校验数字、否定、义务、时态与条款引用回检
6. 双视图交付默认本地语言，可随时展开原文与逐段对齐
7. 反馈学习律师确认/修订进入术语与评测闭环

按内容类型制定不同质量门槛

内容类型转换要求必须锁定人工复核规则
日常消息/任务实时、自然、保留语气与行动项人名、机构、时间、金额低风险可直接发送，允许查看原文
合同与附件逐段对齐、结构与编号不变定义、权利义务、否定、数字、法条关键条款必须双语并列复核
法律意见/审查报告保留不确定性、证据与引用风险等级、法域、法源、修改建议对外发出前由律师确认
仲裁文书底稿术语和格式符合目标机构要求当事人、请求、金额、事实、证据编号仅作为底稿，不自动提交
关键原则：系统不把译文覆盖原文；每次转换都保存原文、译文、术语版本、模型/规则版本、人工修订与确认人。任何高风险内容都能回到原文证据。

04 · Agent engineering

Agent 研发体系

采用“共享底座 + 国家知识包 + 业务 Agent + 人工复核”的组合。先用检索、结构化 workflow 与评测把边界做稳；只有当数据量和误差证据充分时才做专项微调。

Agent 组合

Agent职责主要工具/知识输出禁止事项
任务编排 Agent识别意图、拆分步骤、选择国家与技能、汇总状态任务 schema、权限、案件上下文执行计划、工具事件、待确认项不得越权访问或替代律师作最终决定
双语转换 Agent翻译、术语锁定、双语对齐与一致性回检国家术语库、translation memory原文/译文/差异/置信提示不得改写金额、义务、否定或引用
合同审查 Agent解析、分类、风险扫描、分级、建议与引用国家法源、合同 playbook、风险 taxonomy结构化风险 + 双语报告无依据不得生成确定性结论
合同生成 Agent收集交易要素、选模板、组装条款、一致性检查国家模板、条款库、审批规则双语草案、缺失项、差异说明不得隐藏未确认变量
仲裁辅助 Agent事实抽取、时间线、请求与证据清单、底稿生成经确认的规则库、文书结构与案例结构化案情、双语文书底稿不得自动提交或承诺程序结果
法律研究 Agent限定法域检索、效力判断、规则比较与引用官方/授权法源、版本元数据引用式备忘录、未解决问题不得跨法域混引或引用失效文本
工作助理 Agent邮件摘要、任务抽取、排期、提醒与协作案件、邮件、日历、任务权限草稿、待办、提醒、日程建议未确认不得对外发送或改变关键状态

数据与训练闭环

法源授权来源、版权、有效性、更新时间登记
清洗切分去重、OCR 校正、结构恢复、法域标签
双语对齐条款/句段对齐、术语和实体锁定
专家标注风险、依据、建议、难例与拒答边界
金标评测专家一致性确认，形成不可训练测试集
RAG/微调先检索与 workflow，必要时 LoRA/分类器
线上反馈律师修订→误差归因→回归测试→新版本

训练资产分层

  • 知识库：法律法规、解释、规则、模板及有效性元数据。
  • 训练集：可用于分类、抽取、翻译与建议生成的已授权样本。
  • 金标集：50 份深标合同 + 每国难例，不进入训练。
  • 回归集：500 份去敏合同/文书，版本发布前全量跑测。

Agent 发布门禁

  • 结构抽取、风险召回、引用正确率与双语一致性分开计分。
  • 高风险错误（法域错、金额错、否定翻转、伪造法源）零容忍。
  • 新版本必须比基线提升，且不使已有国家/合同类型回退。
  • 生产输出显示证据、版本、置信提示和人工复核状态。
不是所有问题都用微调解决：法条更新优先进入可版本化知识库；业务步骤优先用可测试 workflow；稳定、重复且有足够高质量标签的误差，才进入微调候选。

05 · Six-country matrix

俄罗斯及中亚五国：独立语料、独立 Agent、独立验收

六国共享技术底座，但不得共享未经区分的法律结论。每个国家单独建立法源清单、术语库、合同风险 playbook、金标集、版本号与当地专家签字记录。

每个国家的标准实施路径

阶段工作包最低产物通过条件
① 法源确认确认官方/授权来源、效力层级、语言版本、更新与使用权限法源登记册 + 更新责任人当地法律专家书面确认
② 语料工程采集、去敏、OCR、清洗、结构恢复、法域/语言/文种标签法规、合同、仲裁与术语四类数据包抽样准确率达到约定阈值
③ Playbook定义合同类别、风险 taxonomy、等级、依据与修改建议国家审查规则 v1 + 典型例两名专家对同一样本结论可复现
④ Agent 适配配置检索、工具、提示词、输出 schema、拒答与人工复核规则国家 package + 自动测试无跨法域混引，高风险强制复核
⑤ 训练/优化对稳定误差做分类器、rerank 或 LoRA 实验；保留对照组模型卡、数据卡、实验记录对金标集显著提升且无回归
⑥ 实案验收当地律师用真实案件盲测，记录接受/修改/拒绝原因验收报告 + 修订集 + 上线限制达到国家门槛后才标记可商用
法源注意：各国官方入口、抓取许可、文本效力和俄文/本地语言版本均须由当地律师在入库前复核；页面中的国家语言与工作包是研发规划，不构成法律意见。

06 · Go-to-market

商业化路径与合作模式

先用真实案件证明“更快、更准、可追溯”，再把交付方法标准化。产品收入与专家服务收入分开，避免客户把软件订阅误解为无限法律服务。

订阅

按组织规模、席位、国家包与功能模块收取年度订阅费。

按量

合同页数、审查份数、Token、OCR/ASR、外部数据查询按量计费。

私有化

一次性实施费 + 年度维保费 + 可选本地模型/知识库更新费。

专家服务

语料整理、playbook 建设、国家法律顾问和专项验收单独报价。

高优先级风险

法律与翻译质量

措施：国家隔离、证据引用、双专家金标、难例集、强制复核、模型回滚。触发：高风险漏检、伪造引用、否定/金额翻转。

数据权利与跨境安全

措施：授权登记、去敏、最小权限、客户隔离、日志、删除/导出流程。触发：来源不明、数据越权、无法证明删除。

六国范围过宽

措施：俄罗斯/哈萨克斯坦先行，其余国家按数据和专家到位程度分级上线，不以一个总准确率掩盖国家差异。

发布和演示失败

措施：独立演示环境、离线样例、至少 5 次完整彩排、现场故障脚本与回滚负责人。

月度决策会：每月底只做四种决策——通过、限范围通过、延期、停止。任何国家 Agent 未通过法源/金标门禁，不得因论坛或销售压力标记为可商用。

15 · Partnership asks

合作方支持清单与启动条件

合作方提供的不是“泛泛建议”，而是可排期、可验收的专家时间、数据与客户资源。以下条件越早锁定，六个月计划越可信。

数据与授权

  • 按国家提供可合法用于研发的去敏合同、仲裁文书与审查意见。
  • 明确数据使用、保存、训练、删除、对外展示与案例引用边界。
  • 指定每国法源与更新责任人，确认语言版本与效力。

专家与验收

  • 至少 1 名俄语法律总负责人，协调六国当地顾问。
  • 每周固定评审时段，完成金标、难例和版本签字。
  • 论坛对外案例必须获得客户和专家书面授权。

客户与市场

  • 10 月前确认 3–5 家种子律所及真实试用联系人。
  • 共同定义付费试点价格、成功标准与退出条款。
  • 11 月论坛提供案例、发言人、媒体和线索跟进机制。

项目治理

  • 双方各指定一名可决策负责人，重大范围变更书面确认。
  • 建立数据、法律质量、技术、商业四个工作组。
  • 确认知识产权归属、成果使用权与保密安排。
